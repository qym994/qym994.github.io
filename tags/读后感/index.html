Proxy​

const p = new Proxy(target,{handler})
Proxy构造函数接收两个变量，第一个参数是被代理的对象，第二个参数是拦截器，也就是一组trap
通过在拦截器里添加第三个参数建立代理对象和副作用函数的联系，例如：
get(target,key,receiver)receiver可以理解为指定函数调用中的this

常规对象，异质对象​

常规对象：常规内部方法为ECMA规范10.1.x定义，内部方法[[Call]]在10.2.1定义，[[Construct]]在10.2.2定义
异质对象：不满足以上规定的对象
Proxy显然是异质对象，异质对象体现了js的多态性，在创建代理对象时如果密钥设定拦截器，则代理对象内部方法会调用原始对象的内部方法完成操作，实质上，创建代理对象时指定的handler也就是拦截函数时用来自定义代理对象本身的内部方法和行为

如何代理一个Object？​

拦截其属性的增删改查操作，利用Proxy对象与常规对象内部方法对应的处理器函数进行拦截，封装在相应的拦截器中
考虑从原型中继承属性的情况：
讨论属性存在于实例还是原型上，假如只存在于原型上，触发副作用函数时会触发两次，实例和原型各一次
判断拦截函数传入参数receiver是否是参数target的代理对象
拦截函数中receiver永远是操作的那个代理对象

数组和集合类型代理​

原理上都大致相同，使用和原始对象内部方法一一对应的Proxy处理器函数拦截操作
具体等待读完数组和集合类型相关内容后补充

响应系统的作用与实现​

借助于代理对象Proxy,在trap的get方法和set方法中分别封装了track和trigger
在get中track收集副作用函数，在set中trigger触发副作用函数
effect函数用来注册副作用函数
响应系统的可调度性：trigger触发副作用函数执行时有能力决定函数执行的时机次数和方式(scheduler)

计算属性computed和懒执行lazy​

懒加载lazy：在effect函数中，options中添加lazy属性，可以指定副作用函数不立即执行
计算属性computed：结合调度器scheduler完成懒执行的有缓存机制(dirty的值)的副作用函数

watch​

实质是对effect函数的二次封装
在响应式数据发生变化后，执行watch本身创建effect的调度器函数schedule，以及两个属性flush和immediate
竞态问题(比如异步发起请求覆盖的问题)：在watch中传入第三个参数onInvalidate注册过期回调函数，在过期回调中处理问题
